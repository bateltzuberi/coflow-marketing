# Coflow — Marketing site

The public-facing marketing site for [Coflow](https://coflow.social) — the all-in-one workspace for social media agencies.

Lives on the apex domain (`coflow.social`). The product app lives at `app.coflow.social` in the [`bateltzuberi/sma`](https://github.com/bateltzuberi/sma) repo.

## Why this repo exists separately

Marketing and product are separated at the domain + repo level because:

1. **SEO authority stays on the apex.** Subdomains are treated as separate sites by Google. The marketing site owns `coflow.social`; the product app sits at `app.coflow.social`. All inbound links hit the apex.
2. **Performance.** Marketing pages are static / ISR, served from Netlify CDN. No middleware, no Supabase round-trips, no authenticated app server in the request path. Fast Core Web Vitals = better rankings.
3. **Independent deploys.** Content edits don't redeploy the product; product changes don't redeploy content.
4. **Separation of concerns.** Marketing doesn't need database access, auth, or product dependencies.

## Routes

```
/                                   Homepage
/pricing                            Plans
/features/content-calendar          Flagship feature page (custom copy)
/features/[slug]                    Other features (driven by lib/content.ts)
/vs/planable                        Flagship comparison (custom copy)
/vs/[slug]                          Other competitors (driven by lib/content.ts)
/for/[slug]                         Persona pages (3 variants)
/blog                               Blog index (posts land in phase 2)
/sitemap.xml                        Auto-generated from site + content data
/robots.txt                         Generated by app/robots.ts
```

Full routes reachable from nav + footer. Every URL returns a real, indexable page with metadata, canonical, OG, JSON-LD.

## SEO

All SEO helpers live in [`lib/seo.tsx`](lib/seo.tsx):

- `buildMetadata()` — consistent per-page title/description/canonical/OG/Twitter
- `organizationJsonLd()` — emitted in the root layout
- `softwareApplicationJsonLd()` — emitted on the homepage
- `breadcrumbsJsonLd()` — emitted on deep pages
- `faqJsonLd()` — emitted wherever we ship a FAQ block (home, pricing, features, vs)

Site-level content (features list, competitors, personas) lives in [`lib/site.ts`](lib/site.ts). Per-competitor, per-feature, and per-persona copy lives in [`lib/content.ts`](lib/content.ts).

## Phase 2 — what to build next

When you return to this, the biggest ranking leverage left on the table is:

1. **Real blog posts** (MDX). Start with: "The social media agency pricing playbook", "Why we priced per-brand not per-seat", "Client approval workflows that don't leak into WhatsApp". Each is its own long-tail-keyword magnet.
2. **OG images.** Put a `/og/default.png` in `/public/og/` (1200×630) and per-page variants for the homepage, pricing, each `/vs/*` page. Pages currently fall back to the default.
3. **Favicon + social icons.** Drop `favicon.ico` + `icon.png` + `apple-icon.png` into `/app/` (Next.js App Router will wire them automatically).
4. **Real pricing numbers** once agreed. Coded as placeholders in `app/pricing/page.tsx`.
5. **Real migration docs** for the Scale plan — linked from /vs pages.
6. **A short intro video / product screenshots** embedded on the homepage (Loom or similar).
7. **Internal linking audit** once blog posts land — every blog post should link to the feature page it exemplifies.
8. **Analytics.** Add Plausible or Fathom (privacy-friendly — keeps the site fast and avoids a cookie banner).
9. **`/changelog`** once the product starts shipping publicly — great for SEO long-tail and for trust.

## Deploy

Netlify, pointed at this repo.

1. Create a new Netlify site → connect the GitHub repo.
2. Build command: `npm run build` (already in `netlify.toml`).
3. Publish directory: `.next`.
4. Install the `@netlify/plugin-nextjs` plugin (auto-installs from `netlify.toml`).
5. In domain settings, point `coflow.social` (apex) at the Netlify site.
6. Verify `/sitemap.xml` and `/robots.txt` return 200.
7. Submit `https://coflow.social/sitemap.xml` to Google Search Console and Bing Webmaster.
